Synthese - Decision Tree + selection KBest

Le modele Decision Tree avec selection KBest a ete optimise avec le F1-score comme metrique principale.
Les meilleurs parametres sont : criterion=entropy, max_depth=7, min_samples_leaf=1, min_samples_split=2.
Accuracy train : `0.8175`, accuracy test : `0.8008`. Les scores train et test sont tres proches, ce qui indique que l'overfitting est limite.
F1-score test : `0.7234`, precision : `0.7640`, recall : `0.6869`. Le recall est moyen : le modele retrouve une partie importante des survivants, mais en manque encore plusieurs.
F1-score moyen en validation croisee : `0.6985` avec un ecart-type de `0.0271`. L'ecart-type du F1-score en validation croisee est faible, ce qui indique une stabilite correcte entre les folds.
La matrice de confusion donne TN=141, FP=21, FN=31, TP=68.

Points a retenir :
- Metrique principale : F1-score.
- Le score d'accuracy est utilise comme indicateur secondaire.
- Le recall indique la capacite a retrouver les survivants.
- La custom accuracy accepte les faux negatifs selon le choix d'interpretation du projet.
